title: Using npx package
---

The **npx Project Creation Tool** simplifies the setup of new Nano Service projects, enabling developers to focus on writing backend logic rather than configuring boilerplate code.

## 🚀 Quick Project Initialization
Run the following command to create a new project:
```bash
npx nanoservice-cli create project
```

### Interactive Setup
The CLI will guide you step by step:

<Steps>
  <Step title="Choose a Template">
  Select from the available templates: **HTTP API** (A pre-configured Node.js + Express project) or **
Scheduled Jobs** (CRON-style workflows)
  </Step>
  <Step title="Project Name">
   Provide a name for your project.
  </Step>
  <Step title="Setup Complete">
   The project will be cloned into the selected directory with all dependencies ready to install.
  </Step>
</Steps>

### Example Workflow

<Steps>
  <Step title="Run the command">
  ```bash
npx nanoservice-cli create project
```
  </Step>
  <Step title="Follow the interactive prompts">
   ```plaintext
    Select a template: HTTP API
    Project Name: my-api-project
    Install dependencies and start coding:
   ```
  </Step>
  <Step title="Install dependencies and start coding">
   ```bash
    cd my-api-project
    npm install
    npm run dev
    ```
  </Step>
</Steps>

## 📁 Project Structure
Each project comes with a clean and organized folder structure, ready for development:

```plaintext
my-api-project/
├── src/
│   ├── workflows/
│   └── nodes/
├── package.json
└── README.md
```
