## **Common Utilities in Shared**
### **1. Logging (`logger.ts`)**  
Provides consistent logging for the entire framework.  
Example:
```typescript
export function logInfo(message: string) {
  console.log(`[INFO]: ${message}`);
}

export function logError(message: string, error?: Error) {
  console.error(`[ERROR]: ${message}`, error || '');
}
```

### **2. Error Handling (`errorHandler.ts`)**  
Handles global errors across Runner, Nodes, and Triggers.  
Example:
```typescript
export class NanoserviceError extends Error {
  constructor(message: string, public code?: number) {
    super(message);
    this.name = 'NanoserviceError';
  }
}
```

### **3. Configuration Loader (`config.ts`)**  
Loads common settings from environment variables or files.  
Example:
```typescript
export const config = {
  maxRetries: process.env.MAX_RETRIES || 3,
  logLevel: process.env.LOG_LEVEL || 'info',
};
```

## **Adding Features or Fixing Bugs**
### **1. Create a Feature Branch**  
```bash
git checkout -b feature/improve-logging
```

### **2. Make Your Changes**
Modify the relevant files inside `/core/shared/src/`.  
Example: **Adding a debug log level in `logger.ts`**  
```typescript
export function logDebug(message: string) {
  if (process.env.LOG_LEVEL === 'debug') {
    console.debug(`[DEBUG]: ${message}`);
  }
}
```

### **3. Test Your Changes**  
Run unit tests to ensure no regressions:  
```bash
npm test
```

### **4. Submit a Pull Request**  
Push your changes and open a PR on GitHub.  
```bash
git push origin feature/improve-logging
```
In the PR description, include:  
- **What was changed?**  
- **Why was it changed?**  
- **How was it tested?**
