<Tip>Trigger is an event or condition that starts the execution of a workflow. Triggers are defined at the top of a [Workflow JSON](/docs/d/core-concepts/workflows.mdx).</Tip>

# Available Triggers
### HTTP
The HTTP trigger listens for HTTP requests, making it ideal for creating APIs or responding to webhooks.
#### Workflow setup
```json GET Example
"trigger": {
    "http": {
        "method": "GET",
        "path": "/example",
        "accept": "application/json"
    }
}
```
<ResponseField name="method" type="GET POST PATCH PUT DELETE">
    HTTP method for the trigger
</ResponseField>
<ResponseField name="path" type="string">
    Endpoint path where the workflow is accessible.
</ResponseField>
<ResponseField name="accept" type="string">
    Specifies the acceptable response content type (e.g., `application/json`)
</ResponseField>
