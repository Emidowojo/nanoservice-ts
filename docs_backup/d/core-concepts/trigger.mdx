<Tip>Trigger is an event or condition that starts the execution of a workflow. Triggers are defined at the top of a [Workflow JSON](/docs/d/core-concepts/workflows).</Tip>

# Available Triggers
### HTTP
The HTTP trigger listens for HTTP requests, making it ideal for creating APIs or responding to webhooks.
#### Workflow setup
```json GET Example
"trigger": {
    "http": {
        "method": "GET",
        "path": "/example",
        "accept": "application/json"
    }
}
```
<ResponseField name="method" type="GET POST PATCH PUT DELETE">
    HTTP method for the trigger
</ResponseField>
<ResponseField name="path" type="string">
    Endpoint path where the workflow is accessible.
</ResponseField>
<ResponseField name="accept" type="string">
    Specifies the acceptable response content type (e.g., `application/json`)
</ResponseField>
#### CTX
```json CTX Structure
{
    "id": "",
    "config": {},
    "request": {},
    "response": {},
    "error": {},
    "logger": ""
}
```
<ResponseField name="id" type="string">
    Request ID autogenerated for each trigger invocation
</ResponseField>
<ResponseField name="config" type="Object">
    Includes all the nodes configuration added in the workflow.
</ResponseField>
<ResponseField name="request" type="Object">
    An http request object containing headers, body, and query parameters. It follow [http.IncomingMessage](https://nodejs.org/api/http.html#class-httpincomingmessage) interface.
</ResponseField>
<ResponseField name="response" type="Object">
    An http response object containing headers, body, and query parameters. It follow [http.Outgoingmessage](https://nodejs.org/api/http.html#class-httpoutgoingmessage) interface.
</ResponseField>
<ResponseField name="error" type="Object">
    An error object containing error message
</ResponseField>
<ResponseField name="logger" type="DefaultLogger">
    An instance of internal logger object. For more information, check [Logger](/docs/ref/shared/src/type-aliases/LoggerContext)
</ResponseField>

# Custom Triggers

If you want to extend the functionality via custom triggers, check out our contribution guidelines to learn how to create and share your [custom triggers](/docs/c/extensions/triggers) with the community.
